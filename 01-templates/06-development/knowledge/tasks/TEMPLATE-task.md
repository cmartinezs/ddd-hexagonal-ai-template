# Task Template

## Purpose

Template for recording development tasks with lifecycle tracking.

> **Note**: This is a template. Adapt to your workflow.

---

## Structure

```markdown
# Task: T-NNN-title

## Summary
One-line description.

## Context
Background and motivation.

## Proposal
What needs to be done.

## Impact
Modules, migrations, documentation affected.

## State
⬜ Registered | 🔍 In Analysis | 📋 Planned | 🔵 In Development | 🔄 In Review | ✅ Completed

## Relationships
| Type | Reference |
|------|-----------|
| blocking | T-XXX |
| enables | T-XXX |

## Verification
How to verify completion.
```

---

## Task States

| State | Emoji | Description |
|-------|-------|-------------|
| Registered | ⬜ | Submitted, not started |
| In Analysis | 🔍 | Being analyzed |
| Planned | 📋 | Ready to start |
| In Development | 🔵 | Currently implementing |
| In Review | 🔄 | Under review |
| Blocked | 🚫 | Blocked by dependency |
| Completed | ✅ | Implemented and verified |
| Pending UI | 🧩 | Waiting for UI integration |
| Change Control | 🛂 | Change control in progress |
| Archived | ⬛ | Archived |

---

## Naming Convention

- **T-NNN** — Task number (e.g., T-001, T-002)
- **F-NNN** — Feature (long-term initiatives)
