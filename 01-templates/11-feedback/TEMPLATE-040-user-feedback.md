# User Feedback

You are a product manager responsible for collecting, analyzing, and acting on user feedback. This template helps you establish channels, organize feedback, and convert insights into product improvements.

**What This Is**: A system for collecting, categorizing, analyzing, and responding to user feedback from multiple channels

**How to Use**: Establish feedback channels (in-app, support tickets, surveys, interviews). Create a feedback log and review process. Track feedback trends and convert high-impact feedback into requirements

**Why It Matters**: User feedback is the signal for what's working and what's broken. Without structured feedback, you build features users don't want and miss problems they're experiencing

**When to Use**: Throughout product lifecycle. Phase 11 (Feedback) is where you formalize and act on feedback

**Owner**: Product Manager

## Feedback Channels

**What This Section Is**: The different ways users can share feedback with you. Multiple channels ensure you hear from different user segments and capture different feedback types.

| Channel | Description | Frequency |
|---------|-------------|-----------|
| **In-app Feedback** | Widget or button | Continuous |
| **Support Tickets** | Customer support | Continuous |
| **Surveys** | NPS, CSAT | Monthly |
| **Interviews** | User interviews | Quarterly |
| **Social** | Twitter, forums | Continuous |
| **Reviews** | App store reviews | Continuous |

## Feedback Collection

**What This Section Is**: How to organize and categorize the feedback you receive. Consistent categorization enables trend analysis and prioritization.

### Feedback Categories

| Category | Description | Priority |
|----------|------------|----------|
| **Bug Reports** | Something doesn't work | High |
| **Feature Requests** | New functionality wanted | Medium |
| **Pain Points** | Friction or frustration | High |
| **Compliments** | What works well | Medium |
| **Suggestions** | Ideas for improvement | Low |

### Feedback Template

Use this template for documenting each piece of feedback:

```markdown
## Feedback: [Brief Title]

**Source**: [Channel]
**Date**: [YYYY-MM-DD]
**User**: [Customer, Segment]
**Priority**: [P1-P4]

### Description
[What the user said/found]

### Context
- Product area: [Area]
- User segment: [Segment]
- Frequency: [One-time/Recurring]

### Suggested Action
[What user wants/needs]

### Impact Assessment
- Affected users: [Number]
- Severity: [High/Medium/Low]
```

## Feedback Analysis

### Analysis Framework

1. **Categorize**: Group by theme
2. **Prioritize**: Impact × frequency
3. **Validate**: Confirm with data
4. **Plan**: Add to roadmap

### Prioritization Matrix

| | Low Frequency | High Frequency |
|---|---|---|
| **High Impact** | Important | Critical |
| **Low Impact** | Backlog | Consider |

### NPS Tracking

| Score | Category | Action |
|-------|----------|--------|
| 9-10 | Promoter | Leverage for referrals |
| 7-8 | Passive | Improve to promoter |
| 0-6 | Detractor | Understand pain points |

## Action Planning

### Feedback Actions

| Category | Action | Timeline |
|----------|--------|----------|
| **Bugs** | Fix immediately | Current sprint |
| **Critical pain** | Prioritize in roadmap | Next sprint |
| **Feature requests** | Evaluate for roadmap | Next planning |
| **Compliments** | Share with team | Monthly |
| **Suggestions** | Evaluate value | Quarterly |

### Feedback Loop

```
Collect → Categorize → Prioritize → Plan → Implement → Communicate
    ↑                                                          │
    └──────────────────────────────────────────────────────────┘
```

## Tips

1. **Listen actively**: Every piece of feedback matters
2. **Close the loop**: Tell users what you did
3. **Look for patterns**: One user = issue, 10 = problem
4. **Prioritize impact**: Focus on highest impact
