# Human Review Checklist

This checklist ensures AI-generated documentation meets quality, consistency, and architectural standards.

## Purpose

Provide a fast, structured way for humans to validate AI outputs without relying on intuition.

---

## 1. Phase Discipline

- [ ] Is the document in the correct phase?
- [ ] Does it respect agnostic vs. technology-specific boundaries?
- [ ] No technologies in phases 1–5?

---

## 2. Traceability

- [ ] Does every major element trace back to previous phases?
- [ ] Are references explicit (not implicit)?
- [ ] Are assumptions clearly stated?

---

## 3. Specificity

- [ ] Is the content project-specific?
- [ ] Does it avoid generic boilerplate language?
- [ ] Does it reflect real constraints?

---

## 4. Architectural Integrity

- [ ] Does it align with DDD principles?
- [ ] Are bounded contexts consistent?
- [ ] Does it respect hexagonal separation?

---

## 5. Internal Consistency

- [ ] No contradictions with existing documents?
- [ ] Terminology consistent with glossary?
- [ ] Naming stable across artifacts?

---

## 6. Completeness

- [ ] All required sections present?
- [ ] No placeholder/template text left?
- [ ] Edge cases considered?

---

## 7. Clarity

- [ ] Easy to understand by another developer?
- [ ] No ambiguous statements?
- [ ] Structure is readable?

---

## 8. Actionability

- [ ] Can this document be used for next phase?
- [ ] Does it enable development/testing/decision-making?

---

## 9. Red Flags (Reject if any)

- [ ] Sounds correct but says nothing
- [ ] Uses buzzwords without meaning
- [ ] Cannot explain how it connects to previous phase
- [ ] Introduces unexplained concepts

---

## Decision

- ✅ Accept
- ⚠ Needs revision
- ❌ Reject
